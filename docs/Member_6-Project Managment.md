## Member 6 – Project Management & Documentation

### Goal
Coordinate the team and maintain **clear documentation**, ensuring the project is organised, structured, and easy to understand.

### Responsibilities
- Team communication and progress tracking  
- Maintain documentation (README, system design, architecture)  
- Ensure repository structure is clear  
- Document individual contributions  

### Files
docs/
README.md

### How It Should Work
- Documentation should explain:
  - System overview  
  - System architecture  
  - Project structure  
  - Team contributions  
  - Running instructions  
  - Testing instructions  
- Keep documentation up-to-date  

### Task List
1. Maintain README and docs  
2. Document system architecture  
3. Track team contributions  
4. Provide instructions for running/testing  
5. Update docs as project evolves  

### Things to Watch Out For
- Documentation becoming outdated  
- Missing explanations of components  
- Inconsistent formatting
